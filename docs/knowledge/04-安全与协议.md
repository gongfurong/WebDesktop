# 安全与协议

## `http://localhost:<port>`

- 常被桌面浏览器视为开发特例
- 常适合本机安装与基础 PWA 行为测试

## `http://<lan-host>:<port>`

- 属于普通网络 HTTP
- 常适合临时浏览
- 通常不适合验证完整安装能力

## `https://<host>`，但服务端实际仍是 HTTP

- 属于协议不匹配
- 页面访问通常直接失败

## `https://<host>` + 不可信证书

- 页面很多时候仍可继续访问
- 但安装入口、`service worker`、离线能力通常不稳定

## `https://<host>` + 可信证书

- 最适合验证接近正式环境的桌面化能力
- 安装、缓存、离线与浏览器能力最稳定
