# 证书与信任

## 什么时候需要证书

- 当你需要从非本机地址通过 HTTPS 访问，并且希望获得更稳定的安装、缓存和安全上下文能力时

## 什么时候不一定需要证书

- 本机 `localhost` 调试
- 临时浏览页面
- 不追求完整 PWA 安装能力时

## 分发形式

### 传统证书文件

- `.cer`
- `.crt`
- `.pem`

适合：

- Windows
- macOS
- Linux
- 部分 Android 手动导入场景

### `mobileconfig`

适合：

- iOS / iPadOS

## 平台信任路径

### Windows

- 导入系统受信任根证书存储
- 删除与恢复通常应同时考虑：
  - 从信任根移除
  - 清理本地生成的证书材料

### macOS

- 导入系统钥匙串并设置信任
- 反向流程通常包括：
  - 从系统钥匙串移除
  - 清理本地生成的证书材料

### Linux

- 导入系统 CA
- 必要时同步导入 NSS
- 反向流程通常包括：
  - 从系统 CA 移除
  - 必要时从 NSS 移除
  - 清理本地生成的证书材料

### iOS / iPadOS

- 安装描述文件
- 再到证书信任设置中开启完全信任

### Android

- 不建议作为默认产品路径
- 如必须导入，应单独按设备与浏览器验证
