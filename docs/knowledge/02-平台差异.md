# 平台差异

## Windows

- 证书信任路径相对统一
- 本地根证书导入自动化实现成本较低
- 桌面安装入口通常较直观

## macOS

- 证书信任通常通过 Keychain Access 或系统钥匙串
- 可支持桌面站点应用能力
- 安装体验通常不如桌面 Chromium 完整统一

## Linux

- 系统 CA、NSS 和浏览器证书存储差异明显
- 桌面入口位置与行为受发行版、桌面环境、浏览器打包方式影响
- 更适合单独说明，不宜套用 Windows 思路

## iOS / iPadOS

- 主安装路径通常是 Safari 的“共享 -> 添加到主屏幕”
- 不等同于桌面 Chromium 的标准安装弹窗
- 证书分发更适合 `mobileconfig`

## Android

- 更接近标准 PWA 模型
- 主路径通常是浏览器原生安装入口
- 证书导入不建议作为默认用户路径
