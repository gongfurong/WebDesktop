# 术语表

## 网页桌面化

指把 Web 应用以更接近桌面应用或主屏幕应用的方式交付给用户，包括桌面图标、主屏幕图标、独立窗口和相关安装入口。

## PWA

渐进式 Web 应用。通常依赖 `manifest`、`service worker`、安全上下文和浏览器安装能力实现更接近应用的体验。

## 站点应用窗口

浏览器把一个网站以独立窗口方式打开的运行形态。它不一定等于完整 PWA，但用户体验上可能非常接近。

## 安全上下文

浏览器认可的可信运行环境。它通常决定 `service worker`、安装入口和部分高级 Web 能力是否可用。

## 可信 HTTPS

指当前 HTTPS 站点不仅使用了 TLS，而且证书链也被浏览器或系统信任。

## 不可信 HTTPS

指服务端虽然提供了 HTTPS，但证书链未被浏览器或系统信任。页面可能仍可访问，但安装与缓存能力通常不稳定。

## 本机访问

当前设备直接访问自己提供的服务，典型形式是 `localhost`。

## 非本机访问

任何不是通过本机特例访问当前服务的方式，例如局域网 IP、主机名、局域网域名或其他网络地址。

## 主路径

在某个平台或浏览器上，最推荐、最稳定、最适合作为默认引导的安装或访问方式。

## 备选路径

主路径不可用时可以采用的替代方案。通常可用，但体验或稳定性不如主路径。

## 不推荐路径

理论上可能可行，但不适合作为默认方案的路径。常见原因包括支持度弱、体验差、依赖过多手动操作或平台差异过大。

## 主屏幕安装

移动端把网页入口添加到主屏幕的能力。iOS / iPadOS 与 Android 的具体实现和入口不同。

## 独立窗口

页面不是在普通标签页里打开，而是以站点应用或主屏幕应用形态单独运行。

## 证书分发

把根证书或信任材料提供给用户或设备导入的过程。常见形式包括 `.cer`、`.crt`、`.pem`、`.mobileconfig`。

## `mobileconfig`

Apple 体系中的配置描述文件格式，适合分发证书和配置项，常用于 iOS / iPadOS 的证书安装流程。

## NSS

部分 Linux 浏览器或工具链使用的证书数据库体系。某些 Linux 浏览器环境下，除了系统 CA 之外，还需要处理 NSS 存储。
