# 用户使用指南

## 适用对象

- 最终用户
- 演示人员
- 手动验证安装路径的人

## 桌面端安装

推荐浏览器：

- Chrome
- Edge
- Brave

通常路径：

1. 打开页面
2. 点击浏览器地址栏或菜单中的安装入口
3. 确认安装
4. 从桌面或系统应用入口打开

## iOS / iPadOS 安装

推荐浏览器：

- Safari

通常路径：

1. 用 Safari 打开页面
2. 点击“共享”
3. 选择“添加到主屏幕”

## Android 安装

推荐浏览器：

- Chrome
- Edge

通常路径：

1. 如果页面上有“安装到主屏幕”，直接点击
2. 如果没有，打开浏览器菜单
3. 选择“添加到主屏幕”或“安装应用”

## 遇到不安全提示怎么办

- 如果只是临时访问页面，可以继续访问
- 如果希望获得更稳定安装能力，最好使用可信 HTTPS
- iOS / iPadOS 如需减少不安全提示，可安装系统信任证书

## 证书安装与卸载

按平台分别处理：

- Windows：证书管理器
- macOS：Keychain Access
- Linux：系统 CA / NSS
- iOS / iPadOS：VPN 与设备管理、证书信任设置

## 如何手动测试示例项目

### 启动

进入示例目录后运行：

```bash
node server.js
```

如果要测试本地 HTTPS，再先执行证书生成步骤。

### 停止

如果服务在当前终端前台运行，通常直接按 `Ctrl + C` 即可停止。

### 基本测试顺序

1. 启动服务
2. 在本机推荐浏览器中打开页面
3. 确认页面和游戏能正常显示
4. 确认安装提示或平台提示符合预期
5. 如果需要，再继续测移动端和 HTTPS 场景

### 桌面端手动测试

1. 在 Chrome、Edge 或 Brave 中打开页面
2. 查看是否有安装入口
3. 完成安装后，再从桌面或系统入口重新打开
4. 确认页面显示为已安装状态

### iOS / iPadOS 手动测试

1. 用 Safari 打开页面
2. 通过“共享 -> 添加到主屏幕”完成安装
3. 从主屏幕入口再次打开
4. 如需测试证书，再按证书说明单独执行

### Android 手动测试

1. 用推荐浏览器打开页面
2. 优先尝试原生安装入口
3. 如果没有，再走浏览器菜单中的安装路径
4. 从主屏幕入口再次打开

### 如果页面看起来还是旧版本

如果你已经改了代码，但浏览器里仍然像旧版本，或者服务停止后页面仍然能打开，通常是旧的 Service Worker 和站点缓存仍在生效。

推荐顺序：

1. 先启动当前最新版服务
2. 打开页面右上角 `设置`
3. 点击 `清理本地缓存并刷新`
4. 如果仍有问题，再用浏览器开发者工具手动清理站点缓存
