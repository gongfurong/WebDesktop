# AI 禁止事项

## 禁止事项

- 不要把 `localhost` 特例当成所有网络地址都成立的规则
- 不要把“页面能打开”误写成“安装能力完整可用”
- 不要把 iOS / iPadOS 的“添加到主屏幕”误写成标准浏览器安装弹窗
- 不要把某个平台的证书导入流程直接复制到另一个平台
- 不要把临时本地证书方案写成最终面向普通用户的正式发布方案
- 不要把弱支持浏览器包装成推荐主路径
- 不要把固定端口、固定 IP、固定设备型号写进通用方案文档

## 常见误判

- 误判：所有非本机访问都可以直接复用 `localhost` 的安装结论
- 误判：iOS / iPadOS 上只要能添加到主屏幕，就等于证书已信任
- 误判：Android 也应主推证书安装
- 误判：Linux 可以按 Windows 的方式统一自动化
- 误判：浏览器标签页里应该总能检测系统是否已安装图标
